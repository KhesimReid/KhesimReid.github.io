EventLite

Eventlite is a website built as part of the Software Engineering 2 course unit at the University of Manchester. Eventlite is a website for displaying events and their information and includes a differentiation between event attendee views and functionality, and event organizer views and functionality. The goal of this piece of coursework was to introduce us to the ‘Model View Controller’ (MVC) method of building web applications. In carrying out the project we acted as a team of software developers who were tasked weekly with implementing new feature specifications proposed by the would-be clients, such that everything except for the landing page - which we were given creative freedom for - was built to the specifications given.

Languages & Technologies Used: Java, JavaScript, HTML, CSS, Thymeleaf, Spring, Eclipse, Git, Mapbox API, Twitter API

ChatApp!

‘ChatApp!’ is a messaging application developed for Android devices, which was built in order to explore and experiment with data transmissions, by sending text and multimedia messages through a proxy server to a bot – both of which were provided by the University of Manchester. The development of this application involved the implementation of bit error correction and packet replacement, to combat issues which can occur in mobile systems and data transmissions. This project also included working with sound and image processing in order to transmit the multimedia messages and correct discrepancies in the transmitted data. This was also my first time working in Android development.

Languages & Technologies Used: Java, Android Studio

Marketshare

Marketshare is a hackathon project built at the Durhack 2018 hackathon held at Durham University. Marketshare is a website built with the intention of allowing beginner investors to share ideas with friends, track the performance of their stock portfolios and follow people who they view as mentors. Given that we only had 24 hours to come up with a project idea and build it, we only managed to complete a working user profile page. For this project, the team of 4 divided into two sub teams; one working exclusively on backend functionality and the other working on the frontend design. I worked in the backend sub team and was responsible for writing code which got the stock data and ran calculations on it, while my teammate connected the database and set up the page controllers. This was my first experience working with JavaScript.

OnWord

OnWord is a website built to allow users to create crossword puzzles and solve them in a competitive and collaborative environment. The target market for this website was senior citizens, as we hoped it would help in including them in the digital age, as technology advances. We aimed to provide them with an online community in order to combat the isolation of senior citizens. In consideration of this, the website was built with a simple user interface, utilising large icons and text. Furthermore, the colour decisions were made in consideration of making the website accessible to persons afflicted with colour blindness. The About Page, User Profile Page and Contact Us Page were created primarily by me and I assisted in all other parts of the website, except for the creating of the crossword grid and storing of crossword puzzles.

Languages & Technologies Used: HTML, CSS, PHP, SQL, Git
